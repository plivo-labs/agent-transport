# Agent Endpoint

Rust library for SIP calling with Python and TypeScript bindings.
See SPEC.md for full specification.

## Project Structure

Cargo workspace with four crates:

- `crates/pjsua-sys/` — Raw FFI bindings generated by bindgen from pjsua.h
- `crates/agent-endpoint/` — Safe Rust API (SipEndpoint, AudioFrame, events)
- `crates/agent-endpoint-python/` — PyO3 bindings (maturin build)
- `crates/agent-endpoint-node/` — napi-rs bindings (npm package)

## Build

### Prerequisites — install prebuilt SIP library

macOS:
```bash
brew install pjproject pkg-config
```

Debian/Ubuntu:
```bash
sudo apt-get install libpjproject-dev pkg-config clang
```

CentOS/RHEL (enable EPEL first):
```bash
sudo dnf install epel-release
sudo dnf install pjproject-devel pkg-config clang
```

Alpine:
```bash
apk add pjproject-dev pkgconf clang
```

### Building

```bash
cargo build
```

The `pjsua-sys` build.rs uses pkg-config to find the system-installed SIP library. No compilation
from source needed — it just generates Rust FFI bindings via bindgen from the installed headers.

### Python binding

```bash
cd crates/agent-endpoint-python
maturin develop  # dev install
maturin build    # build wheel
```

### Node.js binding

```bash
cd crates/agent-endpoint-node
npm install
npm run build
```

## Key Conventions

- Use the C API (pjsua.h) for FFI — bindgen works with C only
- All SIP stack calls must be properly synchronized
- Audio frames are int16 PCM, 48kHz mono (matching LiveKit AudioFrame format)
- SIP callbacks dispatch events through crossbeam channels to avoid blocking
- Link against system-installed SIP library via pkg-config (prebuilt packages)
- bindgen generates bindings at build time from system headers

## Testing

```bash
cargo test                    # Rust unit tests

```

## Architecture Notes

- The SIP stack has a conference bridge; we create a null audio device + custom media port
- Custom media port's get_frame()/put_frame() bridge SIP audio to our AudioFrame
- SIP callbacks (on_incoming_call, on_call_state, etc.) send EndpointEvents through channels
- The Rust API is synchronous; Python/TS bindings wrap in async
- The SIP stack is initialized once globally — SipEndpoint uses interior mutability
